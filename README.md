## Purpose

This project exists to test the capabilities of Kiro as an AI-powered development tool. The goal was to evaluate how far an AI IDE can go in generating a non-trivial, multi-layer .NET application from a structured spec — including architecture, implementation, and tests — without human intervention on the code.

## Generation Approach

- **Tool:** Kiro AI IDE
- **Method:** Spec-driven development — a requirements document, a design document, and a task list were created first, then the AI implemented all tasks sequentially
- **Testing:** Both unit tests (xUnit) and property-based tests (FsCheck) were generated alongside the code as part of the spec

## What It Does

Simulates an AI-driven technical support workflow for a virtual company. The system:

1. Receives support emails via REST API
2. Classifies them using an LLM (code-related or out-of-scope)
3. Routes to the correct team (Team A or Team B) based on the affected application
4. Assigns the right agent (Backend Developer, Frontend Developer, or QA Engineer)
5. Resolves the bug through AI-powered root cause analysis
6. Generates a simulated code fix and pull request

The company has two teams, each with three AI agents. All agents run as Akka.NET actors, all reasoning flows through Microsoft Semantic Kernel + OpenAI ChatGPT.

## Configuration

Set the `OPENAI_API_KEY` environment variable. Model name is configurable in `appsettings.json` (defaults to `gpt-4o-mini`). Teams, agents, and visualization toggle are also in `appsettings.json`.
