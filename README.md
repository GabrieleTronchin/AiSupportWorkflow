# AI Support Workflow

An AI-driven technical support workflow experiment built with .NET 10.0. This project simulates the full lifecycle of a support request — from email intake to automated code fix generation — using LLM-powered agents.

> This entire codebase was generated by AI using [Kiro](https://kiro.dev) as a spec-driven development experiment. Nothing was manually written, reviewed, or audited. Use at your own risk.

## What It Does

The system receives support emails via a REST API, classifies them using an LLM, and routes each issue to the correct team and agent based on the affected application and issue category. An AI agent then performs root cause analysis and generates a simulated code fix as a pull request. The full pipeline runs through an actor-based architecture where each agent operates as an independent actor.

## Key Technologies

- [Microsoft Semantic Kernel](https://learn.microsoft.com/en-us/semantic-kernel/overview/) — AI orchestration and LLM integration
- [Akka.NET](https://getakka.net/) — Actor model for agent lifecycle, supervision, and message passing
- [OpenAI](https://platform.openai.com/docs) — LLM provider (GPT-4o-mini by default)

## Getting Started

1. Clone the repository.

2. Create the local configuration file with your API key:

   `src/AiSupportWorkflow.Presentation/appsettings.Development.json`

   ```json
   {
     "LlmProvider": {
       "ApiKey": "YOUR_API_KEY_HERE",
       "Provider": "OpenAI",
       "ModelName": "gpt-4o-mini"
     }
   }
   ```

   This file is git-ignored and will not be committed.

3. Run the project:

   ```bash
   dotnet run --project src/AiSupportWorkflow.Presentation
   ```
